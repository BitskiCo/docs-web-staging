Token balances will return the assets of a given wallet for the following types
of primitives:

- fungible (ERC20)
- semi-fungible (ERC1155)
- non-fungible (ERC721)

Right now only EVM chains, Ethereum and Polygon, are supported.

### Query Parameters

<ParamField query="chainId" type="integer" default="1" required>
  Set the network to get data from. For EVM chains, see a list of available
  chains here.
</ParamField>

<ParamField query="address" type="string" required>
  The wallet address to fetch.
</ParamField>

<ParamField query="nfts" type="boolean">
If true, will fetch NFT (ERC721 and ERC1155) balances. If false, will only fetch ERC20 assets.

</ParamField>

<ParamField query="contractAddresses" type="array of string">
  The contract addresses to get NFTs from (up to a max of 20 contracts). If
  used, only the NFTs that were minted from the addresses will be returned. Only
  used if nfts=true is passed in as well. Values should be passed as a comma
  separated list, e.g. `contractAddresses=0xabc,0x123`
</ParamField>

<ParamField query="fetchNftMetadata" type="boolean">
  If true, will fetch metadata for NFTs. Only used if `nfts=true` is passed in
  as well.
</ParamField>

<ParamField query="forceRefresh" type="boolean">
  If true, the cache for the asset will be cleared and set by the next request.
</ParamField>

### Response

    <Expandable title ="show objects" >

        <ResponseField name="address" type="string" required>
        The address that owns the token.
        </ResponseField>

        <ResponseField name="amoung" type="string" required>
        Number of tokens owner has in their possesion.
        </ResponseField>

        <ResponseField name="value" type="string" >
        Value of token in USD (used with ERC20 tokens).
        </ResponseField>

        <ResponseField name="blockNumber" type="string" >
        Block number when last evaluated.
        </ResponseField>

        <ResponseField name="token" type="object" required>
        Bitski's standard token model
        </ResponseField>

    </Expandable>

</ParamField>
